Project: Secure Cloud Storage with Azure Key Vault & Encryption

Objective

  • Encrypt sensitive data at rest using Customer-Managed Keys (CMK) with Azure Key Vault.
  • Enforce secure data in transit using HTTPS and TLS 1.2+.
  • Implement least-privilege access using Azure Role-Based Access Control (RBAC).
  • Enable diagnostic logging and trigger alerts for key access anomalies.

Steps Taken

  1. User and Identity Setup: Established user accounts and groups in Azure Active Directory to mirror department roles. These were essential for applying granular access controls through RBAC, ensuring that each user only had access to the resources required for their role. This segmentation minimizes insider threats and strengthens identity governance.
  2. Key Vault Configuration: Created a Key Vault with soft-delete and purge protection to ensure recoverability and compliance. Within it, a Customer-Managed Key (CMK) named StorageEncryptionKey was generated to control storage encryption. Giving encryption control to the organization enhances regulatory compliance and auditing capability.
  3. Managed Identity Deployment: Set up a user-assigned managed identity named StorageEncryptionIdentity. This identity was given key permissions in Key Vault, enabling secure, passwordless interaction with encryption services. It streamlines access management while maintaining tight security standards.
  4. Storage Account Setup: Deployed a secure Azure Storage Account configured with encryption using the CMK. Network access was restricted to HTTPS only, enforcing data protection in transit. Binding the managed identity to the account ensured encryption activities were securely authorized by identity.
  5. Logging and Monitoring: Diagnostic settings were applied to both the storage account and Key Vault to forward logs to a Log Analytics Workspace. This centralization of logs allows for historical queries, security auditing, and correlation of access patterns or anomalies. It builds a foundation for observability.
  6. Alerting: Azure Monitor was used to create alerts for abnormal activities such as failed key access or repeated operations. Alerts were configured to trigger emails via Action Groups. These real-time notifications allow for rapid incident response and monitoring of sensitive operations.

Outcome

The project successfully demonstrated how to implement secure, monitored, and compliant cloud storage using native Azure capabilities. All configurations adhered to best practices for encryption, identity management, and alerting.
