Project: Secure Cloud Storage with Azure Key Vault & Encryption

Objective

  • Encrypt sensitive data at rest using Azure Key Vault and Customer-Managed Keys (CMK).
  • Secure data in transit with enforced HTTPS connections.
  • Control access with role-based access control (RBAC) using Azure AD.
  • Enable diagnostic logging and configure alerts for key usage and anomalies.

Tools Used

  • Azure Active Directory (Microsoft Entra ID)
  • Azure Key Vault
  • Azure Role-Based Access Control (RBAC)
  • User-Assigned Managed Identity
  • Azure Monitor
  • Kusto Query Language (KQL)
  • Log Analytics Workspace

Steps Taken

  1. Identity Setup:
    Created Azure AD users and assigned them to role-specific security groups. Used RBAC to enforce the principle of least privilege for access to both Key Vault and the Storage Account.
  2. Key Vault and Encryption Configuration:
    Deployed Azure Key Vault with soft delete and purge protection enabled. Generated a CMK (Customer-Managed Key) and created a User-Assigned Managed Identity to enable the Storage Account to access the key securely.
  3. Storage Deployment:
    Created a General Purpose v2 Storage Account with HTTPS enforced. Linked the encryption settings to the Key Vault key using the managed identity, enabling encryption at rest with CMK.
  4. Monitoring Configuration:
    Enabled classic diagnostic settings on both the Storage Account and Key Vault. Configured logs to be sent to Log Analytics Workspace.
  5. Alerting and Threat Detection:
    Created alerts using Azure Monitor for key usage, failed access attempts, high-volume blob downloads, and off-hours access by writing custom KQL queries. These alerts were tied to an action group for real-time email notifications.

Outcome

This project successfully demonstrates how to secure cloud-based storage using Azure-native tools with full encryption, monitoring, and role-based access. All configurations followed Microsoft security best practices.
