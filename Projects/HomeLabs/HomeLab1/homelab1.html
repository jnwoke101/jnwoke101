Project: Secure Cloud Storage with Azure Key Vault & Encryption

Objective

  • Encrypt sensitive data at rest using Customer-Managed Keys (CMK) with Azure Key Vault.
  • Enforce secure data in transit using HTTPS and TLS 1.2+.
  • Implement least-privilege access using Azure Role-Based Access Control (RBAC).
  • Enable diagnostic logging and trigger alerts for key access anomalies.

Tools Used

Steps Taken

  1. User and Identity Setup: Created user accounts for IT, HR, and executives. Grouped users into Azure AD security groups and applied appropriate roles through RBAC for granular access.
  2. Key Vault Configuration: Deployed a secure Key Vault with soft-delete and purge protection enabled. Created a CMK (StorageEncryptionKey) and assigned a user-assigned managed identity access to it with 'Get', 'Wrap', and 'Unwrap' permissions.
  3. Managed Identity Deployment: Created a user-assigned managed identity (StorageEncryptionIdentity) and allowed it access to the key vault for encryption operations.
  4. Storage Account Setup: Provisioned a Storage Account (bldynstorage) in East US with HTTPS-only access, enabled blob/file encryption using CMK from the Key Vault, and applied the managed identity.
  5. Logging and Monitoring: Enabled diagnostic settings for the storage account and Key Vault to forward logs to a Log Analytics Workspace. Categories included audit logs and key usage.
  6. Alerting: Created alert rules in Azure Monitor using KQL queries to detect anomalies such as failed access attempts or excessive key operations. Alerts were tied to an Action Group to send email notifications.

Outcome

The project successfully demonstrated how to implement secure, monitored, and compliant cloud storage using native Azure capabilities. All configurations adhered to best practices for encryption, identity management, and alerting.
